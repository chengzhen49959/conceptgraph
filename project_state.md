**Done ‚Äî ingestion pipeline (arq worker + Redis + S3), end to end:**
- parse (**PyMuPDF4LLM** ‚Üí layout-aware Markdown for PDF, multi-column reading order **preserved by rendering each column under its own cropbox** (`_pdf_to_markdown` ‚Äî the engine otherwise y-sorts a table-split two-column page's fragments and scrambles the reading order + merges side-by-side tables; columns detected via `column_boxes`, tiled gap-free) + structural headings, References section stripped, **then layout-engine noise stripped** ‚Äî figure/formula `==> picture ‚Ä¶ intentionally omitted` placeholders + `----- picture text -----` OCR'd chart-axis garble, line-break hyphenation rejoined, bare page numbers + Unicode/blank-run normalised, all marker/line-anchored so no substantive text is touched; direct for MD/txt) ‚Üí chunk (hash-idempotent) ‚Üí embed chunks ‚Üí extract concepts + relations (OpenAI structured output, per chunk, **pure recall** ‚Äî every named concept, salience deferred to the gate) ‚Üí **doc summary** ‚Üí **core-concept gate** ‚Üí **merge/dedup** ‚Üí build edges (relation + **co-occurrence**) ‚Üí **clustering**, with `documents.status` advancing **at every stage** (`pending ‚Üí parsing ‚Üí chunking ‚Üí embedding ‚Üí extracting ‚Üí merging ‚Üí clustering ‚Üí done|failed`) so the frontend can show the live phase, not just a spinner.
- Core-concept gate (reduce-pass salience, `app/ai/select_core.py`): per-chunk extraction is blind to the whole doc, so related-work / bibliography passages each yield noise nodes. Centrality is global, so one document-level LLM call decides it. It judges each candidate against the document's **framing** ‚Äî title + abstract + intro + conclusion, sliced positionally from the parsed text by `app/services/thesis.py` (fixed front/tail char windows, **not** section parsing ‚Äî slicing never fails to return an anchor). The framing replaced the aggregate doc summary as the arbiter: the summary stitches every passage together and surfaces incidental machinery (optimizer, LayerNorm) as if it were the subject; the authors' framing names only the contribution. Each candidate carries `freq` (passage count, weak hint) + **`grounded`** (name/alias literally present in the framing ‚Äî a strong, deterministic core prior, but English-only: concept names are normalised to English while the framing stays in source language, so for non-English docs grounding goes quiet and the model reads the framing directly ‚Äî the prompt is told an absent flag means nothing); the prompt applies an **aboutness** test (is the doc *about* it, not does the doc *depend on* it) and leans **drop** when unsure (precision > recall). Drops are document-scoped (other docs' nodes untouched) and fail-open (refusal/empty/empty-framing ‚Üí keep all, summary as fallback anchor). Only kept concepts are embedded/merged.

- **Merge/dedup is built and hardened** (the core differentiator): alias fast path + ANN + LLM judge + per-workspace lock + ON CONFLICT backstop. A same-form false-merge eval is still TODO before enabling any auto-merge band (skip-the-judge above a high similarity). 
- **The original single-user PRD loop is now complete** ‚Äî RAG Q&A (F8) shipped 2026-06-27 alongside semantic search (F7), and everything upstream (ingest ‚Üí graph) and the collaboration layer are in place. **F18 (MCP external memory + graph App) shipped 2026-06-28**, exposing the core loop to external AI clients. **F15 (Slack) is the sole remaining unbuilt feature.**

- **Cost / deploy:** Render workers need Starter (Free has no workers); Free web spins down ‚Äî use Starter for any demo.
- **Local run gotcha:** the `arq` worker is a *separate* process from the uvicorn API. If only the API runs, uploads enqueue but nothing drains the queue, so documents sit at `pending` forever (looks like a hang, not an error). Start it with `cd backend && uv run arq app.worker.WorkerSettings`. Images are now rejected at upload (415, no OCR).
- **Local run gotcha (frontend, 2026-06-25):** Next 16 blocks `127.0.0.1` as a cross-origin dev host by default, so on `http://127.0.0.1:3000` the page renders but **never hydrates** ‚Äî no event handlers attach, and a form silently falls back to a native GET submit (login "does nothing", no error, no network call). Fixed by adding `allowedDevOrigins: ['127.0.0.1']` to `frontend/next.config.ts` (dev-only; restart `next dev` after). Tell-tale: `next dev` log `‚öÝ Blocked cross-origin request ‚Ä¶ /_next/webpack-hmr from "127.0.0.1"` + a looping `webpack-hmr ‚Ä¶ ERR_INVALID_HTTP_RESPONSE` console error. **Same 127.0.0.1-vs-localhost root cause on the backend:** `cors_allow_origins` only had `http://localhost:3000`, so every API call from `127.0.0.1:3000` failed its CORS preflight (no ACAO header). Fixed in `config.py` by mirroring each configured http origin to its localhost‚Üî127.0.0.1 sibling, so both hosts work; the API's `--reload` picks it up live.
- **Frozen mid-pipeline status (root cause of the "stuck PDF" incident):** the worker advances `status` at phase *boundaries* ‚Äî as of 2026-06-24 this includes `extracting` / `merging` / `clustering` (the finer-granularity fix below), so the long embed‚Üíextract‚Üímerge‚Üícluster tail no longer masquerades as a frozen `embedding`. If the worker process is **hard-killed** mid-job (laptop sleep / closed terminal), the SQLAlchemy `except` never runs, status stays frozen at `embedding` (not `failed`), and nothing re-drives it. arq later *retries* the orphaned job, but a manual re-enqueue on top causes duplicate jobs to contend on the per-workspace `merge-lock` ‚Üí repeated 600 s `job_timeout`s. Mitigations: (1) **client timeout fixed** ‚Äî `AsyncOpenAI(timeout=45)` so a stalled call can't eat the whole 600 s budget (`app/ai/client.py`); **(3) finer status granularity ‚Äî done (2026-06-24):** `worker.py` now sets `extracting` / `merging` / `clustering` across the former blind span, and the frontend documents list + status bar render the live stage label (`DOC_STATUS_LABEL` in `lib/api.ts`, shown in `NavSidebar`/`StatusBar`) instead of an opaque spinner. Still TODO: (2) a startup sweep that resets non-terminal-but-stale docs to `failed`/re-enqueue. **(4) clear *concepts* (not just chunks) so partial-run nodes don't accumulate ‚Äî done (2026-06-26), see the orphan-concept reconciliation entry below.**
