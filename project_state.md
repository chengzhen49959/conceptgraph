**Done ‚Äî ingestion pipeline (arq worker + Redis + S3), end to end:**
- parse (**PyMuPDF4LLM** ‚Üí layout-aware Markdown for PDF, multi-column reading order + structural headings, References section stripped; direct for MD/txt) ‚Üí chunk (hash-idempotent) ‚Üí embed chunks ‚Üí extract concepts + relations (OpenAI structured output, per chunk, high recall) ‚Üí **doc summary** ‚Üí **core-concept gate** ‚Üí **merge/dedup** ‚Üí build edges (relation + **co-occurrence**) ‚Üí **clustering**, with `documents.status` advancing **at every stage** (`pending ‚Üí parsing ‚Üí chunking ‚Üí embedding ‚Üí extracting ‚Üí merging ‚Üí clustering ‚Üí done|failed`) so the frontend can show the live phase, not just a spinner.
- Core-concept gate (reduce-pass salience, `app/ai/select_core.py`): per-chunk extraction is blind to the whole doc, so related-work / bibliography passages each yield noise nodes. Centrality is global, so one document-level LLM call decides it. It judges each candidate against the document's **framing** ‚Äî title + abstract + intro + conclusion, sliced positionally from the parsed text by `app/services/thesis.py` (fixed front/tail char windows, **not** section parsing ‚Äî slicing never fails to return an anchor). The framing replaced the aggregate doc summary as the arbiter: the summary stitches every passage together and surfaces incidental machinery (optimizer, LayerNorm) as if it were the subject; the authors' framing names only the contribution. Each candidate carries `freq` (passage count, weak hint) + **`grounded`** (name/alias literally present in the framing ‚Äî a strong, deterministic core prior, but English-only: concept names are normalised to English while the framing stays in source language, so for non-English docs grounding goes quiet and the model reads the framing directly ‚Äî the prompt is told an absent flag means nothing); the prompt applies an **aboutness** test (is the doc *about* it, not does the doc *depend on* it) and leans **drop** when unsure (precision > recall). Drops are document-scoped (other docs' nodes untouched) and fail-open (refusal/empty/empty-framing ‚Üí keep all, summary as fallback anchor). Only kept concepts are embedded/merged.
- Merge/dedup: exact-name **and alias** fast path (no LLM) ‚Üí pgvector ANN recall ‚Üí `match_concept` LLM judge ‚Üí fold description + re-embed + alias. The whole merge/edge/cluster section runs under a per-workspace Redis lock (no duplicate nodes under concurrency); an `INSERT ‚Ä¶ ON CONFLICT(workspace_id, lower(name))` is the race backstop. The lock TTL is **tied to `_JOB_TIMEOUT`** (`worker.py`): a shorter TTL expired mid-merge under a multi-PDF batch (`Cannot release a lock that's no longer owned`), which also let a second job grab the lock mid-merge and break F4 ‚Äî fixed by pinning both to the same constant. `max_tries=3` so one transient Aurora/DNS blip mid-write under batch load retries instead of permanently failing the doc.
- **Global dedup sweep** (`app/services/dedup_sweep.py`, arq job `dedup_sweep_workspace`): repair pass for the greedy incremental merge. Re-blocks every concept pair by exact cosine, re-runs the `match_concept` judge, and **merges two existing nodes** (re-points mentions/edges/aliases incl. `kind='cooccur'`, folds the description, deletes the loser; union-find for transitive A‚â°B‚â°C). Same merge lock; `dry_run` previews without writing; idempotent. Verified end-to-end (`test/dedup_sweep_smoke.py`: plants a split duplicate ‚Üí sweep folds it, no mention/edge loss, second run is a no-op).
- **Co-occurrence edges** (`worker.py` Phase 2): concept pairs sharing a chunk get an undirected `kind='cooccur'` edge, weight = co-mention count accumulated across chunks/documents. Hidden from the graph view (`routers/graph.py` filters `kind='relation'`), used only to densify the cluster graph. On the attention.md smoke: 26 concepts ‚Üí 40 relation + 128 co-occur edges (‚âà39% of the complete graph, weight avg 1.27 / max 4), lifting clustering from 3 leaves / 0 parents to 5 leaves / 2 parents.
- Doc summary: per-chunk summaries aggregated into one document summary, embedded, stored on `documents.summary` ‚Äî **search only**; the core-concept gate now anchors on the document framing (`app/services/thesis.py`), not this summary (it falls back to the summary only when the framing is empty).
- Clustering: two-level Leiden over the weighted graph ‚Äî relation **and** co-occurrence edges (igraph/leidenalg, networkx-Louvain fallback) ‚Üí leaf `cluster_id` + `parent_id` hierarchy; orphan concepts (no edge) rescued by embedding-kNN; LLM labels each cluster; recomputed per workspace after each ingest.
- AI adapter layer (`app/ai/`): embeddings, extract, core-concept gate, merge judge, labels, summarize. OpenAI-only, backend-only.
